智慧医院智能导诊系统（Smart Hospital Triage System）

项目类型：AI + 医疗信息化  

技术栈：Spring Boot · Qt · Web · LangChain4j · RAG · Qwen-Turbo · MySQL  


📌 项目简介

本项目是一套集智能问答、人工协同、预约挂号于一体的智慧医院导诊平台，旨在解决传统医院导诊效率低、人工压力大、患者就医流程复杂等问题。

系统基于 Agent + RAG 架构，实现了：
• 多轮症状问诊与科室推荐

• AI 智能客服与人工客服无缝切换

• 全流程预约挂号闭环

• 患者情绪识别与智能调度

• 高并发场景下的稳定服务支撑

适用于互联网医院、线下门诊大厅、医院小程序/APP等多种场景。

🧠 核心亮点（面试加分项）

✅ AI Agent 智能导诊

• 基于 LangChain4j 构建 Agent 架构

• LLM Planner：意图识别 + 多步动作规划

• Embedding Router：自动路由至 5 大知识库（FAQ / 医生 / 科室 / 医保 / 导航）

• Tool Calling：支持科室查询、医生排班、预约挂号等工具调用

• 相比传统 RAG，准确率更高、幻觉更少

✅ 多知识库 RAG 检索增强

• 相似度阈值 0.3，Top-K=5

• 检索失败自动触发业务兜底话术

• Prompt 结构化约束，显著降低 AI 幻觉

• 支持知识库热更新（一键刷新向量索引）

✅ 情绪识别 & 智能调度

• 用户情绪评分模型（不耐烦 → 自动转人工）

• 基于情绪权重 + 等待时长的动态排队算法

• 客服技能标签 + 负载均衡，支持 VIP 插队

✅ 全链路隐私保护

• 自动脱敏：手机号、身份证、邮箱、银行卡号

• JWT 鉴权 + 权限隔离（患者 / 客服 / 医生 / 管理员）

• HTTPS + 数据加密存储，符合医疗数据安全规范
